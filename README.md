# Promo_Calculator

## UnityVersion

6000.3.16f1

## Описание

Реализован модульный калькулятор на Unity с использованием:

- Clean Architecture
- MVP (Model-View-Presenter)
- Assembly Definition для разделения модулей

Поддерживаемая математическая операция:

- сложение (`+`)

Реализованный функционал:

- сохранение состояния приложения между сессиями
- хранение истории вычислений
- восстановление последнего введённого выражения после перезапуска
- обработка некорректных выражений
- вывод ошибок через отдельный Notification модуль


# Архитектура

Проект разделён на независимые модули:

Каждый модуль является отдельной Assembly Definition и имеет собственную область ответственности.

## Core

Базовый модуль приложения.

Отвечает за:

- запуск приложения (`EntryPoint`)
- инициализацию систем
- загрузку сцен
- общую систему сохранений

---

## Calculator

Независимый модуль калькулятора.

Особенности:
- не зависит от Core
- может быть перенесён и переиспользован в другом проекте

Основная логика калькулятора разделена на несколько частей:

### ICalculatorService

Интерфейс для взаимодействия с внешними модулями.

### CalculatorProcessor

Отвечает за выполнение вычислений.

Получает выражение пользователя и выбирает подходящую для него математическую операцию через: `ICalculatorOperationProvider`

Добавление новых математических операций возможно без изменения существующей логики.

Для добавления новой операции достаточно реализовать `ICalculatorOperation` и зарегистрировать её в системе.

### ICalculatorStateService

Он отвечает за состояние калькулятора в runtime.

---

## Notification

Независимый модуль уведомлений.

Отвечает за:

- отображение информационных сообщений
- отображение ошибок пользователю

Модуль:

- не зависит от Core
- не зависит от Calculator
- может использоваться в других проектах

### INotificationService

Интерфейс для взаимодействия с внешними модулями.

---

# Система сохранений

В Core реализована общая система сохранения состояния приложения через `SaveService`.

Сохранение выполняется в формате JSON.

Каждый модуль, которому необходимо сохранять состояние, подключается через интерфейс: `ISaveModule`

## Интеграция Calculator с SaveService

Для подключения Calculator к общей системе сохранений используется адаптер: `CalculatorSaveModule`

Он реализует `ISaveModule`  и связывает `SaveService` с `CalculatorStateService`

Таким образом:

- Calculator остаётся независимым модулем
- Core управляет общей системой сохранения
- связь между ними происходит через адаптер

# UI

Для реализации UI используется архитектурный паттерн: MVP

### View
Отвечает только за отображение и считывание пользовательского ввода.

### Presenter
Отвечает за взаимодействие между View и сервисами.
